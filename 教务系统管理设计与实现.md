# 教务系统管理设计与实现
本项目基于 **C语言自研HTTP服务后端 + HTML/CSS前端** 的B/S架构实现，C语言完成核心业务逻辑、数据持久化、数值计算与HTTP请求响应处理，HTML/CSS实现可视化页面交互，覆盖管理员、教师、学生三类角色，完整实现选课管理与成绩管理全流程。

## 一、系统核心数据定义
### 1. 学生信息
每条学生记录包含：学号（唯一主键，10位数字）、姓名、性别、年级、所属专业、联系电话、登录密码（加密存储）。
- 约束：学号全局唯一；初始密码默认为学号后6位，首次登录可修改。

### 2. 教师信息
每条教师记录包含：工号（唯一主键，6位数字）、姓名、性别、所属院系、联系电话、登录密码（加密存储）。
- 约束：工号全局唯一；初始密码默认为工号后6位。

### 3. 管理员信息
每条管理员记录包含：管理员账号（唯一主键）、姓名、登录密码（加密存储）。
- 约束：系统内置1个超级管理员，可新增次级管理员账号。

### 4. 课程信息
每条课程记录包含：课程号（唯一主键）、课程名称、课程类型（必修/选修）、学分、授课教师工号、适用专业列表、上课时段规则、选课人数上限、已选人数、平时成绩占比、期末成绩占比、课程教学大纲、课程状态（未发布/选课中/授课中/已结课）、起止周次、课程起止日期。
- 上课时段规则：支持多时段配置，格式为「每周X 第Y-Z节」，严格遵循四川大学江安校区作息时间。
- 适用专业：支持设置多个专业，仅对应专业学生可查看并选课。
- 约束：课程号全局唯一；已选人数不可超过人数上限；选课开启后，课程号、学分、上课时间、人数上限、适用专业不可修改。

### 5. 选课记录
每条选课记录包含：记录ID、学号、课程号、选课时间、选课状态（已选）。
- 约束：同一学号+同一课程号只能存在1条有效记录；系统不支持退课操作，选课成功后不可撤销。

### 6. 成绩记录
每条成绩记录包含：记录ID、学号、课程号、平时成绩、期末成绩、总评成绩、对应绩点、录入时间、录入教师工号。
- 约束：同一学号+同一课程号只能有1条成绩记录；成绩取值范围为0-100。

### 7. 系统配置信息
包含：学期开学日期、选课开启时间、选课关闭时间、全局系统状态、专业列表。
- 学期开学日期用于自动推算每一周的对应日期范围，实现周次与日期的双向对应。

## 二、管理员功能模块
### (1) 账号信息管理
#### ① 学生账号管理
- 新增学生信息：输入学号、姓名、性别、年级、专业、联系电话，系统自动校验学号唯一性，生成初始密码并加密，按学号升序写入学生数据文件。
  提示：先将数据读入结构体数组，校验通过后插入对应位置，排序后回写文件。
- 修改学生信息：输入学号查询对应学生，可修改姓名、年级、专业、联系方式；支持密码重置（重置为初始密码并加密）。
  提示：遍历数组匹配学号，修改对应字段后重新写入文件。
- 删除学生信息：输入学号查询学生信息，二次确认后删除记录，同步级联删除其选课记录与成绩记录。
  提示：重建新文件，将未删除的记录写入新文件后替换原文件。
- 浏览与查询学生信息：支持按学号、姓名、年级、专业筛选，分页展示所有学生信息。

#### ② 教师账号管理
- 新增、修改、删除、查询教师信息，逻辑与学生账号管理一致，主键为教师工号，支持按院系筛选。

#### ③ 管理员账号管理
- 新增、修改、删除次级管理员账号，支持重置管理员密码。

### (2) 选课周期配置
- 设置选课时间段：输入选课开启时间与关闭时间，写入系统配置文件；系统自动根据系统时间判断当前选课状态。
- 手动控制开关：支持强制开启/关闭选课通道，覆盖时间配置。
- 选课状态监控：查看当前选课阶段状态，统计各课程已选人数、选课完成率。

### (3) 课程全局管理
- 查看所有课程列表：支持按课程类型、授课教师、专业、年级筛选。
- 修改课程信息：可调整课程所有属性，选课进行中修改时间、人数上限、适用专业等核心信息需二次确认。
- 删除课程：无选课记录的课程可直接删除；已有学生选课的课程需先清退学生再删除。

### (4) 系统基础配置
- 设置学期开学日期，自动推算全学期周次与日期的对应关系。
- 维护全校专业列表，供课程设置适用专业时选择。

## 三、教师功能模块
### (1) 课程开设与管理
- 新增课程：填写课程号、课程名称、类型、学分、适用专业、上课时间段、人数上限、平时/期末成绩占比、教学大纲、起止周次，系统自动根据开学日期计算课程起止日期，也支持手动修改；提交后存入课程文件，状态默认为「未发布」。
  提示：校验课程号唯一性，自动关联当前登录教师工号作为授课教师。
- 修改课程信息：课程未发布时可修改所有信息；选课开启后仅可修改教学大纲等非核心信息。
- 查看我的课程：列表展示本人开设的所有课程，显示课程状态、已选人数/上限、适用专业。

### (2) 选课名单管理
- 查看课程选课名单：选择对应课程，展示所有已选学生的学号、姓名、年级、专业、联系方式。
- 支持按学号、姓名搜索学生，支持导出选课名单为文本格式。

### (3) 成绩录入与管理
- 课程结课操作：选课阶段结束后，教师可对课程执行「结课」操作，课程状态变为「已结课」，进入成绩录入阶段。
- 成绩录入：按学生逐条录入平时成绩、期末成绩，系统自动按预设占比计算总评成绩，并换算为4分制绩点，写入成绩文件。
  提示：总评成绩 = 平时成绩 × 平时占比 + 期末成绩 × 期末占比，结果保留1位小数。
- 成绩修改：结课后可修改学生成绩，修改后自动重新计算总评与绩点。
- 成绩统计：自动计算所授课程的平均分、最高分、最低分、及格率、各分数段人数分布。

## 四、学生功能模块
### (1) 个人信息管理
- 查看个人基本信息：展示学号、姓名、性别、年级、专业、联系方式。
- 修改个人信息：可修改联系电话。
- 修改登录密码：验证原密码后设置新密码，加密后更新到文件。

### (2) 选课中心
- 浏览可选课程：选课阶段开启后，展示所有本人专业可选择的课程列表，显示课程号、名称、授课教师、学分、课程类型、上课时间、已选/人数上限、考核占比。
- 查看课程详情：点击课程可查看完整信息，包括教学大纲、考核方式、教师信息、起止周次与起止日期。
- 选课操作：选择课程后，系统自动校验三项规则：① 上课时间是否与已选课程冲突；② 是否已选该课程；③ 课程是否有名额；校验通过则选课成功，更新课程已选人数与选课记录。
  提示：遍历学生已选课程，逐一比对上课时段与周次范围，存在重叠则判定为时间冲突；选课成功后不可退课。
- 我的课表：以周课表形式展示所有已选课程的上课时间，直观查看时间安排。

### (3) 成绩查询与统计
- 已出成绩列表：展示所有已结课课程的课程名、学分、课程类型、总评成绩、对应绩点。
- 成绩明细查看：点击课程可查看平时成绩、期末成绩、占比、总评、绩点详情。
- 成绩自动计算：系统自动统计并展示四项核心指标：
  ① 加权平均成绩 = Σ(课程总评成绩 × 课程学分) / Σ所有课程学分
  ② 加权平均学分绩点（GPA） = Σ(课程绩点 × 课程学分) / Σ所有课程学分
  ③ 必修加权平均成绩 = Σ(必修课程成绩 × 必修课程学分) / Σ必修课程学分
  ④ 必修加权平均绩点 = Σ(必修课程绩点 × 必修课程学分) / Σ必修课程学分

### (4) AI智能学习助手
- 成绩分析触发：学生点击「AI成绩分析」按钮，系统自动拉取该生所有已出成绩与统计数据。
- 数据请求：后端将内置系统提示词 + 学生完整成绩数据，通过HTTP请求调用第三方AI大模型API。
- 结果展示：获取AI返回的分析建议后，渲染到页面展示，内容包括成绩画像、优势学科分析、薄弱点提示、学习改进建议。
  提示：系统内置固定提示词模板，自动填充学生成绩数据，无需学生手动输入；支持重复调用生成分析。

|      | 第4节 | 11:10-11:55 | 第二大节 |

### 2. 选课冲突判定规则
- 时间冲突：两门课程的上课「星期+节次」存在重叠，且周次范围存在重叠，则判定为时间冲突，不可同时选择。
- 专业限制：学生专业不在课程的适用专业列表中，不可选择该课程。
- 人数限制：课程已选人数达到上限时，关闭选课通道。
- 重复选课：同一学生不可重复选择同一门课程。
- 阶段限制：仅选课阶段开启时，可进行选课操作；选课关闭后仅可查看。
- 不可退课：选课成功后全程不可退课，无退课功能。

### 3. 4分制绩点换算标准
采用国内高校通用细分分段标准：

| 总评成绩区间 | 对应绩点 |
|--------------|----------|
| 90 ~ 100     | 4.0      |
| 85 ~ 89      | 3.7      |
| 82 ~ 84      | 3.3      |
| 78 ~ 81      | 3.0      |
| 75 ~ 77      | 2.7      |
| 72 ~ 74      | 2.3      |
| 68 ~ 71      | 2.0      |
| 64 ~ 67      | 1.5      |
| 60 ~ 63      | 1.0      |
| 0 ~ 59       | 0.0      |

### 4. 周次与日期对应规则
- 管理员设置学期开学日期后，系统自动将开学日期所在的自然周定义为第1教学周，依次推算第2周、第3周……的起止日期。
- 教师设置课程起止周次后，系统自动匹配对应的课程起止日期；同时支持教师手动修改课程起止日期，手动设置优先级高于自动推算。

## 六、技术架构与实现方案
### 1. C语言简易HTTP服务器
- 基于TCP Socket实现，监听指定端口（如8080），支持处理浏览器发起的GET、POST请求。
- 核心流程：接收TCP连接 → 解析HTTP请求行、请求头、表单数据 → 根据URL路径路由到对应业务处理函数 → 执行业务逻辑（文件读写、计算、校验） → 构造HTTP响应头 + HTML页面 → 发送响应 → 关闭连接。
- 支持Cookie实现登录状态保持，区分三类角色的访问权限。

### 2. 前后端交互
- 前端使用纯HTML+CSS构建页面，表单通过POST方式提交数据到后端对应接口。
- 后端处理完成后，直接返回完整的HTML页面，实现页面跳转与数据展示。
- 所有页面共用一套CSS样式，统一教务系统风格，包含登录页、三类角色的功能主页、详情页等。

### 3. AI API调用实现
- C语言封装HTTP客户端函数，支持发送POST请求，携带JSON格式的请求体。
- 内置系统提示词模板，示例：「你是一名专业的学业指导老师，请根据以下学生的成绩数据，给出成绩分析和学习建议，内容包括优势学科、薄弱环节、后续学习规划，语言简洁明了，适合大学生阅读。学生成绩数据如下：{成绩数据}」
- 调用第三方大模型API，将返回的文本内容嵌入HTML页面返回前端。

### 4. 密码加密方案
- 采用MD5哈希算法加密存储密码，不保存明文。
- 登录时将用户输入的密码进行MD5加密，与文件中存储的密文进行比对验证。
- C语言集成轻量级开源MD5算法实现，无需依赖第三方库。

## 七、数据存储设计
所有数据采用二进制文件持久化存储，共7个数据文件，按功能分文件存储，结构体定长存储便于随机读写：
1. `student.dat`：学生信息文件，按学号升序存储。
2. `teacher.dat`：教师信息文件，按工号升序存储。
3. `admin.dat`：管理员信息文件。
4. `course.dat`：课程信息文件，按课程号升序存储。
5. `selection.dat`：选课记录文件。
6. `score.dat`：成绩记录文件。
7. `system_config.dat`：系统配置文件，存储学期信息、选课时间、专业列表等全局配置。
